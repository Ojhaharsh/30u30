Scaled Dot-Product Attention

The core mechanism that makes Transformers work

Attention(Q, K, V) = softmax(QKT / ‚àödk) V
Where Q = Query, K = Key, V = Value, d_k = dimension of keys
Q
Query
"What am I looking for?"
Each position broadcasts a query - a representation of what information it needs from other positions.
K
Key
"What do I contain?"
Each position provides a key - a representation of what information it holds that others might want.
V
Value
"What do I return?"
The actual content that gets passed when a query matches a key. The output is a weighted sum of values.

Why Multiple Heads?

üéØ Different Patterns

Each head can specialize in different types of relationships:

‚Ä¢ Syntactic: Subject-verb agreement
‚Ä¢ Semantic: Word meaning similarity
‚Ä¢ Positional: Nearby words
‚Ä¢ Coreference: Pronouns to nouns

üî¢ Dimension Split

With d_model = 512 and 8 heads:

‚Ä¢ Each head: d_k = 512/8 = 64 dimensions
‚Ä¢ Total params: Same as single-head!
‚Ä¢ Compute: Parallel across heads
‚Ä¢ Output: Concatenate & project

Positional Encoding

Teaching the model about word order using sinusoidal functions

PE(pos, 2i) = sin(pos / 100002i/d)     PE(pos, 2i+1) = cos(pos / 100002i/d)
Different frequencies for different dimensions

üåä Why Sinusoids?

‚Ä¢ Unique encoding for each position
‚Ä¢ Relative positions computable as linear functions
‚Ä¢ Extrapolates to longer sequences than training
‚Ä¢ Bounded values between -1 and 1

üìê Different Frequencies

‚Ä¢ Low dimensions: Slow waves (global position)
‚Ä¢ High dimensions: Fast waves (local details)
‚Ä¢ Combined: Unique fingerprint per position
‚Ä¢ Similar positions ‚Üí Similar encodings
